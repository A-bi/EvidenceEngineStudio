
# Privacy

## Privacy overview

**EvidenceEngineStudio** is designed as a local-first scientific workspace.

The intended workflow is that:
- project organization remains local
- manuscript drafting remains local
- critique remains local
- dataset handling remains local
- most research logic remains local and is based on an API guiding to my website and research engine evidenceengine.eu

Online connectivity is only required when the user intentionally uses the linked **EvidenceEngine** literature search layer.

---

## Local use

EvidenceEngineStudio is intended to operate primarily on the user’s computer.

This means the application is designed for:
- local project management
- local writing workflows
- local critique workflows
- local data inspection
- local analysis review

If you process sensitive, personal, clinical, or regulated data, you are responsible for ensuring that use of the app complies with:

- GDPR
- institutional data protection policies
- ethics and governance requirements
- academic or clinical confidentiality rules
- local laws and regulations

---

## EvidenceEngine as external evidence layer

EvidenceEngineStudio may connect to **EvidenceEngine** when the user actively uses the evidence search layer.

According to the public EvidenceEngine website and privacy policy, EvidenceEngine is described as:

- European
- hosted in Europe
- privacy-first
- using no user accounts
- using no tracking tools
- using no marketing cookies
- not permanently storing search queries

The same policy also states that:
- TinyLlama runs locally on the server
- no personal data is transferred outside the European Union
- technically required logs are shortened or anonymized and retained for a maximum of 7 days
- search queries are processed only temporarily to retrieve and display results

Website: https://evidenceengine.eu

---

## Sources used by EvidenceEngine

The public EvidenceEngine site lists these sources:

- OpenAlex
- PubMed
- Europe PMC
- DOAJ
- arXiv

These are part of the external evidence retrieval layer and are not the same as the local project workspace inside EvidenceEngineStudio.

---

## User responsibility

If you use EvidenceEngineStudio with sensitive material, you should:

- anonymize data whenever possible
- avoid using identifiable personal information unless strictly necessary
- avoid uploading confidential material to public bug reports
- verify institutional approval before using regulated or clinical data
- use synthetic or reduced examples when reporting issues

---

## No guarantee

EvidenceEngineStudio is intended to support scientific organization and reasoning.

It does **not** guarantee:
- correctness of imported data
- correctness of analyses
- correctness of literature interpretation
- correctness of manuscript output
- compliance with journal policies
- compliance with institutional or legal requirements

The user remains responsible for verifying all outputs and ensuring appropriate data protection.
