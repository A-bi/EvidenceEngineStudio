# EvidenceEngineStudio

**EvidenceEngineStudio** is a local-first scientific workspace for building, interrogating, and writing research projects in one structured desktop environment.

It is designed to connect:

**question → hypothesis → evidence → dataset → analysis → critique → manuscript**

instead of scattering those steps across disconnected tools.

---

## What EvidenceEngineStudio does

EvidenceEngineStudio combines multiple parts of the research workflow in one application:

- project design and research framing
- hypothesis structuring and project logic
- evidence linkage and paper management
- local dataset inspection
- statistical analysis and result review
- critique of hypotheses, sources, and data
- manuscript drafting with sections, figures, data attachments, citations, and formulas
- export workflows for further writing and submission

The goal is not just to store notes or run isolated analyses, but to preserve the logic of scientific reasoning across the whole workflow.

---

## Core workflow

### 1. Project design
Create a project, define the research question, and attach working notes.

### 2. Hypothesis layer
Build and refine explicit hypotheses rather than leaving assumptions implicit.

### 3. Evidence layer
Link relevant papers to the active project and keep the literature close to the question it is meant to support.

### 4. Data layer
Import and inspect datasets locally, review structure and variables, and prepare analysis.

### 5. Analysis layer
Run quantitative workflows, review summaries and plots, and reuse outputs in critique and writing.

### 6. Critique layer
Challenge the hypothesis, the evidence base, and the dataset before converting results into strong narrative claims.

### 7. Writing layer
Draft manuscripts with editable sections, citations, formulas, figures, data references, and export tools.

---

## Why this workspace exists

Most research workflows are fragmented:
- literature search in one tool
- notes in another
- statistics in another
- manuscript drafting elsewhere

EvidenceEngineStudio is built to keep those parts connected, so the project question, the linked evidence, the chosen dataset, the analysis output, the critique, and the manuscript remain part of the same research object.

---

## Local-first architecture

EvidenceEngineStudio is intended to run locally on the computer.

The main workspace is designed for:
- local project organization
- local manuscript drafting
- local critique
- local dataset handling
- local analysis review

Online connectivity is only needed when the user actively uses the linked **EvidenceEngine** literature layer.

## EvidenceEngine integration

EvidenceEngineStudio can be used together with **EvidenceEngine** as an external evidence search layer.

- privacy-first

- OpenAlex

## Privacy notes

EvidenceEngine’s public privacy policy states that it:
- uses no user accounts
- uses no tracking tools
- uses no marketing cookies
- does not permanently store search queries
- runs its integrated TinyLlama model locally on the server
- does not transfer personal data outside the European Union

EvidenceEngineStudio itself is intended as a local-first desktop workflow.

If you work with sensitive, clinical, or regulated data, you remain responsible for ensuring compliance with:
- GDPR
- local institutional rules
- research governance
- ethics requirements
- confidentiality obligations

---

## Scientific disclaimer

EvidenceEngineStudio is a research workspace.

It is **not** intended to:
- replace scientific judgment
- replace peer review
- guarantee correctness of analyses or interpretations
- provide binding legal or medical conclusions
- replace supervision, statistical consultation, or methodological review

All project logic, analyses, critique outputs, and manuscript text should be reviewed critically by the user.
